# Training-free Visual Exemplar Few-shot Object Detector
以下要件のObject Detectionタスクを実現する技術方案を提案してください。
## 要件：
1. 数千クラスの検出対象物体があります。主にはおもちゃ、フィギュア、ボトル飲料、日常用品などで構成されます。
2. クラス毎に、クラス名、1〜5枚の参考写真、カテゴリの情報があります。例えば："class_name='Pokemon Treecko', refer_images=['001.jpg','002.jpg'], category='toy'"。但し、クラス名は番号のようなもので、必ずしも物体を判明できない場合があります。また、参考写真は物体の各側面の写真であります。
3. クエリ画像は上記数千クラスの中の数クラスの物体が混在する画像です。クエリ画像と一緒に、検出したいクラス名（１〜数クラス）が指定されます。
4. 期待する結果は、クエリ画像の中の指定クラスのInstanceのバウンディングボックスです。
5. クラスの追加・削除も頻繁に発生するため、できるだけTraining-freeの方案が望ましいです。
6. 検出精度はスピードより優先します。
7. 実行環境はNVIDIA Tesla T4 GPU 1枚が搭載するマシンです。
また、ご提案と一緒に、以下の問題に対する回答を含めてください。
## 問題：
1. 例えばあるクラスの物体の正面と背面は全く異なっていて、かつ背面は他のクラスの背面と同じの場合は、どう対応しますか？さらに言えば、クラスを定義する際にまたは参考写真を用意する際にどうすればいいですか？
2. もしクエリ画像の解像度は640x480の場合は推論時間はおおよそどのぐらいになりますか？1600x1200の場合は？
